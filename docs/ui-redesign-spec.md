# Arkiv UI Redesign Spec

This file has been superseded by the rebuild guide in [`docs/rebuild/`](./rebuild/README.md).

Use the rebuild guide as the source of truth for the current `/app` redesign. The old version of this file described an earlier sample-mode and proof-card direction that no longer matches the product target.

Current north star:

```txt
Select memories -> Review what the agent receives -> Copy for agent
```

Current product frame:

- Product name: `Arkiv Context`
- Left panel: `Memories`
- Main panel: `Portable agent context`
- Proof surface: `Proof details`

Key rule:

Proof remains central to Arkiv's value, but the default UI should not expose protocol detail as the primary product surface. Full evidence belongs behind `Proof details`.
